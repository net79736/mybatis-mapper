알림톡 사전조사

① 카카오 서버 (템플릿 방식) - 가장 일반적
알림톡은 스팸 방지를 위해 카카오의 사전 승인을 받아야 합니다. 
그래서 문장 전체를 서버에서 만들어 보내는 게 아니라, 미리 등록된 템플릿 코드를 호출합니다.
관리: 카카오 채널 관리자 센터에 메시지 틀을 등록 (예: ORDER_CONFIRM)
서버 데이터: DB나 코드에서는 ORDER_CONFIRM이라는 코드값과 그 안에 들어갈 **변수 데이터(이름, 가격, 날짜 등)**만 관리합니다

② 자체 DB (기록 및 관리용)
서버에서 발송한 후에는 "누구에게 어떤 템플릿으로 보냈는지"를 DB에 저장합니다.
Table 구조 예시:
```sql
send_log_id (PK)
template_code: WELCOME_MSG
replacement_json: {"name": "홍길동", "item": "맥북"} (가변 데이터)
status: SUCCESS / FAIL
```

3. 알림톡 '템플릿' 관리
알림톡은 아무 내용이나 보낼 수 없고, 미리 **승인받은 틀(Template)**이 있어야 합니다.
관리 장소: **'카카오톡 채널 관리자 센터'**에서 관리합니다.
"귀하의 상품이 [배송상태] 입니다"처럼 변수 처리된 문구들을 여기서 등록하고 수정합니다.
